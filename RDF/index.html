AA sinbolikoa vs AA estatistikoa

AA sinbolikoa: aurretik eraikitako eredua aplikatzen da. Eredu hori adituek sortu dute eta ezagutza esplizitua sinboloetan (logikan) kodetzen du

AA estatistikoa: ereduak datuetatik ikasten du modu (erdi)autonomoan prozesu probabilistikoen bidez

Arrazoiketa

AA sinbolikoa: logikoa, determinista. Azalgarritasun osoa

AA estatistikoa: hurbilketa, probabilistikoa. Azalgarritasun baxua

Datuak

AA sinbolikoa: datu gutxiago behar ditu baina zarataren aurrean ahula

AA estatistikoa: kalitate handiko datu asko behar ditu, zarataren aurrean sendoagoa

Eskakizunak

AA sinbolikoa: domeinuko adituak

AA estatistikoa: datuak eta konputazio-gaitasun handia (GPUak)

Triple Store-ak

  • Datu eredua: Triple Store guztiek berdina kargatzen dute

Triple Store-ak internet existitzen zenean sortu ziren (NoSQL!):

  • Guztiek HTTP(S) SPARQL bidez funtzionalitate bera eskaintzen dute

W3C estandarra garapenean

UPSERT (UPDATE INSERT)

SQLko eragiketa: eguneratzen du existitzen bada, txertatzen du ez bada

INSERT INTO maskotak (id, izena, kolorea)

VALUES (2, "Fluffy", "beltza")

ON DUPLICATE KEY UPDATE kolorea = VALUES(kolorea);

Helburua: errenkada baterako balio eguneratu bakarra mantentzea

Oinarritua: Stratos Kontopoulos (Medium)

RDFn datuak kargatzea gehigarria da

Datuak kargatzean, hirukoitza berriak gehitzen dira

Adibidea: ex:kolorea "beltza" gehitzeak ez du ex:kolorea "marroia" ezabatzen

Emaitza: baliabideak hainbat balio izan ditzake propietate berarentzat

"Balio bakar"-aren semantikarik gabe

RDF ereduak propietate bakoitzeko hainbat balio onartzen ditu baliabide batean

owl:FunctionalProperty-rekin (OWL) ere, murrizketa logikoa da: Triple Store-ak ez du automatikoki ezabatzen

Triple Store-ek onartu, ohartarazi (SHACL) edo inferitu (OWL) dezakete; ez dute automatikoki ezabatzen

Open World Assumption (OWA)

RDFn (eta OWLn), datu bat falta izateak ez du esan nahi faltsua edo zaharkitua denik (Informaziorik ez izateak ez du ukapena inplikatzen, informazioa beste nonbaiten edo etorkizunean egon daitekeelako interneten)

Balio bat ordezkatzeko, ezabaketa esplizitua adierazi behar da

Aukerak: SPARQL Update (DELETE/INSERT) edo grafo osoa ordezkatu

Linked Data

Web sarean datu egituratuak argitaratzea, makinak lotu eta interpreta ditzaten

Web estandarretan oinarritua: URI, HTTP(S), RDF, SPARQL

Tim Berners-Lee-k (Web-aren asmatzaileak) proposatua 2006an

Linked Data-ren printzipioak

  1. URIak erabili baliabideak identifikatzeko
  2. HTTP(S) URIak erabili baliabide horietara sartzeko
  3. Baliabide batera sartzean, bere datuak estandarrak erabiliz eman (RDF, SPARQL)
  4. Beste URIetarako estekak jaso, baliabide gehiago aurkitzeko

https://www.w3.org/DesignIssues/LinkedData.html

Linked Data: Eduki-negoziaketa

URI batek bezeroaren arabera adierazpen desberdinak itzul ditzake

Web nabigatzailea → HTML (gizakientzat)

RDF bezeroa → RDF/XML, Turtle, JSON-LD (makinentzat)

HTTP Content Negotiation bidez (Accept goiburua)
