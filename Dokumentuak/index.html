MongoDB

Dokumentuetan oinarritutako NoSQL datu-basea: datuak JSON dokumentu gisa gordetzen dira

Ez du eskema finkorik (schemaless)

Tresnak

  • mongod — datu basearen zerbitzaria
  • mongosh — shell interaktiboa komandoak exekutatzeko
  • MongoDB Compass — interfaze grafikoa kolekzioak aztertzeko
  • MongoDB Atlas — kudeaturiko cloud zerbitzua

BSON: Binary JSON

MongoDB-k dokumentuak barnean BSON formatuan gordetzen ditu

JSON-aren adierazpen binarioa, biltegirako eraginkorragoa

BSON: Binary JSON

JSON-ek ez dituen datatype osagarriak onartzen ditu:

  • ObjectId — 12 byteko identifikatzaile bakarra
  • Date — milisegundoko doitasuna duten datak
  • NumberInt, NumberLong, NumberDecimal
  • BinData — datu binarioak
  • Regex — adierazpen erregularrak

Alderapen operadoreak

db.alumni.find({ edad: { $gt: 21 } })



db.alumni.find({ edad: { $gte: 20, $lte: 23 } })

$gt, $gte, $lt, $lte, $eq, $ne, $in, $nin

Logika operadoreak

db.alumni.find({

})



db.alumni.find({

  $or: [

})

$and, $or, $not, $nor

Objektuen array-etan kontsultak

    { asignatura: "SO", nota: 7.0 }

  ]

db.alumni.find({ "notas.nota": { $gt: 7 } })

db.alumni.find({

$exists eta $type

// "becado" eremua DUTEN dokumentuak bilatu:

db.alumni.find({ becado: { $exists: true } })



// "telefono" eremua ez DUTEN dokumentuak bilatu:

db.alumni.find({ telefono: { $exists: false } })



// "edad" zenbaki motatakoa den dokumentuak bilatu:

db.alumni.find({ edad: { $type: "number" } })



// "nombre" string bat den dokumentuak bilatu:

db.alumni.find({ nombre: { $type: "string" } })

Aggregation Pipeline

Etapetan (stages) datuak prozesatzeko sistema

Dokumentuak transformazioen kate bat zeharkatzen dute

Datuak iragaztea, eraldatzea, taldekatzea eta aberastea pausoz pauso ahalbidetzen du

db.kolekzioa.aggregate([
  { $etapa1: { ... } },
  { $etapa2: { ... } },
  { $etapa3: { ... } }
])

Etapa baten irteera hurrengoarena da

Pipeline-ren etapa nagusiak

Etapa Deskribapena
$match Dokumentuak iragazi
$group Taldekatu eta kalkulatu
$project Eremuak aukeratu/aldatu
$sort Emaitzak ordenatu
$limit Emaitzak mugatu
$unwind Array-ak deskonposatu
$lookup Kolekzioak batzea

$match: iragaztea

db.alumni.aggregate([

  { $match: { edad: { $gt: 20 } } }

])



// Iragazkiaren sintaxia find()-n berdin da

// $match ahalik eta lehenengo joan beharko litzateke murrizteko

// hurrengo etapetara pasatzen diren dokumentuen kopurua

$group: taldekatu eta kalkulatu

"pedidos" kolekzio berria, pedidos.json kargatu



db.pedidos.aggregate([

  {

    $group: {

      _id: "$cliente",

      total: { $sum: "$precio" },

      num_pedidos: { $sum: 1 }

    }

  }

])

$group funtzioak

MongoDB Deskribapena
$sum Balioak batu edo zenbatu
$avg Batezbestekoa
$min Gutxieneko balioa
$max Gehieneko balioa
$first Taldearen lehenengo balioa
$push Balio guztien array-a

Pipeline osoa: adibidea

db.pedidos.aggregate([

  { $match: { precio: { $gt: 30 } } },            // WHERE

  { $group: {                            // GROUP BY

      _id: "$cliente",

      total: { $sum: "$precio" }

  }},

  { $match: { total: { $gt: 200 } } },        // HAVING

  { $sort: { total: -1 } }                    // ORDER BY

])

Pipeline-ak kontsulta konplexuak pausoz pauso eraikitzea ahalbidetzen du

$project: eremuak aldatu

db.alumni.aggregate([

  {

    $project: {

      nombre: 1,

      email_mayus: { $toUpper: "$email" },

      anio_nacimiento: {

        $subtract: [2026, "$edad"]

      },

      _id: 0

    }

  }

])

$unwind: array-ak deskonposatu

// Datua: { nombre: "Jon", notas: [{asig: "ABD", nota: 7.5},

//                                {asig: "SI", nota: 8.0}] }

// $unwind array-aren eremu bakoitzerako dokumentua sortzen du:



db.alumni.aggregate([

  { $unwind: "$notas" }

])

// Emaitza:

// { nombre: "Jon", notas: { asig: "ABD", nota: 7.5 } }

// { nombre: "Jon", notas: { asig: "SI", nota: 8.0 } }



// Geroago ikasgaian $group egiteko erabilgarria:

db.alumni.aggregate([

  { $unwind: "$notas" },

  { $group: {

      _id: "$notas.asignatura",

      media: { $avg: "$notas.nota" }

  }}

])

$lookup: kolekzioak batzea

db.pedidos.aggregate([

   {

    $lookup: {

         from: "alumni",

         localField: "cliente",

         foreignField: "nombre",

         as: "alumniMatch"

     }

   }

])

MongoDB-n eskema diseinua

MongoDB schemaless bada ere, eskema diseinua oinarrizkoa da

Gako-galdera: txertatu ala erreferentziatu?

  • Txertatu: erlazionatutako datuak dokumentuaren barruan sartu
  • Erreferentziatu: erlazionatutako dokumentuaren _id gorde

Aukera atzipen-patroietan, tamainan eta eguneratze-maiztasunean oinarritzen da

Txertatzea (desnormalizatzea)

// Informazio guztia dokumentu bakarrean

{

  nombre: "Javier Cansado",

  direccion: {

    calle: "Gran Vía 1",

    ciudad: "Bilbao"

  },

  pedidos: [

    { producto: "Monitor", precio: 250, fecha: "2026-01-15" },

    { producto: "Ratón", precio: 25, fecha: "2026-02-20" }

  ]

}

Abantailak: irakurketa oso azkarra (kontsulta bakarra)

Mugak: datuen bikoizketa eta dokumentu handiak

Erreferentziatzea (normalizatzea)

// "clientes" kolekzioa

{

  _id: ObjectId("aaa..."),

  nombre: "Javier Cansado",

  direccion: { calle: "Gran Vía 1", ciudad: "Bilbao" }

}



// "pedidos" kolekzioa

{

  cliente_id: ObjectId("aaa..."),

  producto: "Monitor",

  precio: 250,

  fecha: "2026-01-15"

}

Abantailak: bikoizketarik gabe, dokumentu txikiagoak

Mugak: $lookup behar du (motelagoa) eta kontsulta gehiago

Noiz txertatu? Noiz erreferentziatu?

Irizpidea Txertatu Erreferentzia
Erlazioa 1:1, 1: gutxi 1: asko, N:M
Datuak elkarrekin irakurtzen dira Bai Ez beti
Datuak asko aldatzen dira Ez Bai
Subdatu neurria Txikia/mugatua Handia/mugagabea
Lehentasuna Irakurketa abiadura Koherentzia

MongoDB-n indizeak

Indizerik gabe, MongoDB-k dokumentu guztiak aztertu behar ditu (collection scan)

Indize batek dokumentuak aurkitzea ahalbidetzen du, kolekzio osoa eskaneatu gabe

MongoDB-k automatikoki _id gainean indize bat sortzen du

Edozein eremuren gainean edo eremuen konbinazioetan indizeak sor daitezke

Indizeak sortu eta kudeatu

// Kargatu ehu.alumni.json ehu.alumni1000 kolekzioan



// Existitzen diren indizeak ikusi:

db.alumni1000.getIndexes()



// Indize sinplea sortu:

db.alumni1000.createIndex({ email: 1 })   // 1 = goraka; -1 = behera



// Indize konposatua (eremu anitzetan):

db.alumni1000.createIndex({ nombre: 1, edad: -1 })



// Indize bakarra (ez du bikoiztua baimentzen):

db.alumni1000.createIndex({ email: 1 }, { unique: true })



// Indizea ezabatu:

db.alumni1000.dropIndex({ email: 1 })

explain(): kontsultak aztertu

db.alumni1000.find({ email: "alumno0680@ikasle.ehu.eus" }).explain("executionStats")

// COLLSCAN: indizerik gabe, kolekzio osoa eskaneatzen du

// IXSCAN: indizearekin, zuzenean bilatzen du

Kontsultetan maiz erabiltzen diren eremuetan indizeak sortu

Indizeek irakurketak bizkortzen dituzte, baina idazketak moteldu

Eskema egiaztapena

MongoDB schemaless bada ere, balidazio aukerakoa defini daiteke

JSON Schema erabil daiteke dokumentuetan arauak definitzeko

db.createCollection("ehu-schema", {

  validator: {

    $jsonSchema: {

      bsonType: "object",

      required: ["nombre", "email", "edad"],

      properties: {

        nombre: { bsonType: "string",

                   description: "string bat izan behar du" },

        email:  { bsonType: "string",

                   pattern: "^.+@.+$" },

        edad:   { bsonType: "int",

                   minimum: 16, maximum: 99 }

      }

    }

  }

})

Egiaztapena: mailak

"strict" maila (lehenespenez): insert eta update eragiketetan balioztatzen du

"moderate" maila: dokumentu berriak eta lehendik baliozkoak diren dokumentuak balioztatzen ditu

Eskema malgukortasuna eta datuen integritatea orekatzea ahalbidetzen du

collMod_moderate, collMod_strict, productos_ejemplo, invalid_productos
