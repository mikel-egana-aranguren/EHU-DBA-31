Emaitzak aldagai baten balioaren arabera multzokatu

SELECT ?hiria (COUNT(?museo) AS ?museoKop)

WHERE {

  GRAPH <urn:count> {

    ?museo rdf:type dbpedia:museum ;

    gip_prop:kokalekua ?hiria .

  }

}

GROUP BY ?hiria

ORDER BY DESC(?museoKop)

HAVING

Taldeak filtratu agregazioaren ondoren

SELECT ?hiria (COUNT(?museo) AS ?museoKop)

WHERE {

  GRAPH <urn:count> {

    ?museo rdf:type dbpedia:museum ;

    gip_prop:kokalekua ?hiria .

  }

}

GROUP BY ?hiria

HAVING (COUNT(?museo) > 1)

ORDER BY DESC(?museoKop)

BIND

Balioak aldagaiei esleitu kontsultaren barruan

SELECT ?person ?fullName

WHERE {

  GRAPH <urn:name-surname>{

    ?person foaf:firstName ?first ;

    foaf:lastName ?last .

  }

BIND(CONCAT(?first, " ", ?last) AS ?fullName)

}

MINUS

Patroi batekin bat datozen emaitzak baztertu: museo guztiak, 300 obra baino gehiago dituzten museoak kenduta

SELECT ?museo ?obrak

WHERE {

  ?museo rdf:type dbpedia:museum .

  MINUS {

    ?museo gip_prop:kopurua ?obrak .

    FILTER (?obrak > 300)

  }

}

NOT EXISTS

Patroi baten existentzia ezaren arabera filtratu

EXISTS vs MINUS

EXISTS/NOT EXISTS: filtroa, ez ditu binding-ak ezabatzen

MINUS: soluzio osoak ezabatzen ditu

NOT EXISTS eraginkorragoa da balioztatzerako

MINUS erabilgarriagoa da patroi konplexuen bazterketarako

Property Paths

^: alderantzizkoa (ex:parent^ seme-alabak aurkitzen ditu)

*: zero edo gehiago (ex:knows* ezagunen sarea)

+: bat edo gehiago (ex:subClassOf+ hierarkiarako)

!: edozein erlazio hori izan ezik

Property Paths

?: aukerakoa (zero edo bat)

|: alternatiboa (ex:email|ex:phone)

/: sekuentzia (ex:livesIn/ex:country)

SPARQL 1.1 Service Description

SPARQL endpoint-ei buruzko metadatuak eskaintzen ditu bezeroei laguntzeko:

  • Gaitasunak aurkitu: Zer funtzionalitate onartzen ditu? (agregazioak, property path-ak, ezeztapena, etab.)
  • Dataset eskuragarriak aurkitu: Zer Named Graph daude eskuragarri?
  • Formatu bateragarriak zehaztu: JSON, XML, CSV, ...
  • Inferentzia-erregimenak ikusi: RDFS, OWL, ...

SPARQL 1.1 Federated Query

W3C-ren estandarra (=HTML, !=SQL)

Kontsulta federatuak

Hainbat SPARQL endpoint kontsultatzea ahalbidetzen dute kontsulta bakarrean

Jatorri desberdinetako datuak konbinatu SERVICE hitz gakoa erabiliz

Kontsulta federatuak

INSERT DATA {

  <http://gipuzkoa.eus/resource/bilbo> owl:sameAs <http://dbpedia.org/resource/Bilbao> . 

}
        
SELECT ?pop 

WHERE {

  gip:bilbo owl:sameAs ?town . 

  SERVICE <https://dbpedia.org/sparql/> { 

    ?town <http://dbpedia.org/ontology/populationTotal> ?pop . 

  } 

}

SPARQL 1.1 Graph Store HTTP Protocol

W3C-ren estandarra (=HTML, !=SQL)

Eragiketak

RDF grafoak kudeatu HTTP bidez:

  • GET: Grafoa berreskuratu
  • PUT: Grafoa ordeztu (Suntsitzailea)
  • DELETE: Grafoa ezabatu
  • POST: Tripleak grafoari gehitu

Eragiketak: Parametroak

Oinarrizko endpoint-a: konfigurazioaren arabera

Parametroak:

  • graph=<URI>: Named Graph zehaztu
  • graph gabe: Default Graph-ean du eragina

Content-Type: RDF/XML, Turtle, N-Triples, etab.

Abantailak

  • RESTful: HTTP estandarrean oinarritutako interfaze sinplea
  • Automatizazioa: Script eta CLI tresnekin integratzea
  • SPARQL UPDATE gabe eguneratzeak: INSERT/DELETE-ren alternatiba
  • Bertsioen kudeaketa: Grafoen bertsio osoak ordeztu

Graph Store vs SPARQL UPDATE

Alderdia Graph Store HTTP SPARQL UPDATE
Granularitatea Grafo osoa Triple zehatzak
Interfazea REST/HTTP SPARQL kontsulta
Ohiko erabilera Bertsio osoak ordeztu Aldaketa zehatzak
Formatua RDF (TTL, RDF/XML, etab.) SPARQL INSERT/DELETE
